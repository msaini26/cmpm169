Integrate

After learning from existing techniques and strategies, we can build upon that foundation by combining elements from different sources. This might involve combining elements from different programming languages, using libraries or frameworks in new ways, or bringing together techniques from different fields or disciplines. By integrating and combining elements from different sources, we can create something new and unique.

Once we were able to implement both examples, we ran into some issues on how to combine the two techniques together. We realized that we had to edit how one of the sketches was formatted and turn it into a class instead. This way, we could create an object of the class in the other sketch to draw them on top of each other. Once we reorganized Mansi’s example to follow a class layout, we then ran into another issue. Mansi’s example showed up on top of Rebecca’s design but it doesn’t continuously draw and only shows up as small random dots all over the screen. We then found out that it was because there was a specification of noLoop() inside of the setup of Rebecca’s design, as illustrated in the image below. The example uses this so the generated artwork doesn’t continuously flash a different design and instead, keeps one until you refresh the entire browser. Because of the noLoop(), we had to come up with another method to continuously loop Mansi’s design but only generate Rebecca’s design once. After playing around with the code, we figured out that the easiest solution would be to create a boolean around the display method for Rebecca’s design so it would only draw once. This way, Mansi’s design could continuously draw while Rebecca’s design stays static. After integrating both techniques, the design came out looking very unique. We thought it looked pretty cool but they still looked like two separate designs and not that cohesive. This is when we decided to experiment more with the color schemes as well as drawing styles to make it appear more as one final piece.

Innovate

Finally, we can introduce new or original ideas to our coding process by thinking creatively and pushing the boundaries. This might involve developing entirely new techniques or approaches, or using existing techniques in new and creative ways. By innovating and introducing new ideas, we can create art that is truly groundbreaking.

To first make the color schemes match, we passed in the color palette used from Rebecca’s design into Mansi’s design so that the colors match. We then randomized each of the line colors by picking a random one from the given array color palette as illustrated below. Sometimes, this would lead to every line having the same color but we thought it would be cool that sometimes the lines are monochrome while other times, they have different shades involved.

We then experimented with the initial particle structure, changing how the noise texture appeared on the screen. We played around with the transparency levels to overlay varying opacity levels of colors to indicate depth on the design itself. We also expanded the lower and upper bound for the particle radius, giving way to a smoother feel from the generative art. In addition, we first attempted to increase the maximum number of particles to see what would happen, around 500 as the maximum, but quickly realized that the browser was unable to render that many at once. As a result, after switching the values down to around 150, we noticed that the browser was not only able to generate the particles faster, but the particle trail was more visible within the art piece itself. In addition, then we tried integrating another variation, but changing the initial direction and evolution fo the particles. After experimenting with a variety of fixed combinations for the initial lower bound range of width and height, setting them equal created a unique spiral movement and pattern when generated. Below are a couple of visual examples from the experimentation elements explained here.

Finally, we modified the fourth variation of noise particle clusters even further to change the styling compared to the other cluster variations. First, we attempted to supersize the radius of the cluster particle variation to 130, which looked absolutely incredible. Because of its sheer size, the particles floated like bubbles across the screen compared to the thinner ones snaking across the screen. However, as much as we liked this design variation, we realized that our computers could barely keep up with the amount of particles generated on the screen. As a result, we switched to another option of having an increased radius cluster, but reducing the size to allow for easier particle generation with the computers. We are super excited with how the end result looks with each variation completely unique from the rest. Below are images of the experimentation we described here.
